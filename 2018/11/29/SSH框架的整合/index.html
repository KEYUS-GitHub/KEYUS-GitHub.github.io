SSH框架的整合

Struts2 + Spring + Hibernate的整合开发（基于Maven）

整合理论

  1. Web层Struts2的Action对象交给Spring来管理。
  2. Service层的Service对象交给Spring来管理。
  3. Dao层Hibernate的SessionFactory对象、Session对象以及AOP事务交给Spring来管理。

Maven包管理的核心配置（仅含一些重点配置）

Spring框架的整合

  1. 在resources目录下创建Spring核心配置文件（applicationContext.xml）

  2. 在applicationContext.xml中加入XSD约束（共含有beans、context、aop、tx）

  3. 在web.xml中配置Spring Web项目核心监听器

  4. 在applicationContext.xml中先随便配置一个bean，配置并启动Tomcat运行，如果没有抛出任何异常，证明Spring框架整合成功！

Spring + Struts2框架的整合

  1. 在resources目录下创建Struts2核心配置文件（struts.xml）

  2. 在struts.xml中加入Struts2的核心DTD约束

  3. 在web.xml中配置Struts2核心过滤器

  4. 在struts.xml中随便配置一个Action类，启动Tomcat，如果没有抛出任何异常，证明Struts2框架使用成功！

  5. 在struts.xml中配置Struts2的Action类交给Spring容器统一管理

  6. 在applicationContext.xml中配置Struts2的Action类

  7. 将Struts2的Action类配置项的class属性值修改为对应bean的name值

  8. 为所有的Action类按第6、第7步的方式进行配置。

  9. 启动Tomcat，访问其中一个Action类的URL，如果访问成功，则Spring + Struts2框架的整合成功！
